# Agora — Prerequisites & Setup Checklist

Everything you need before running the app, in order of priority.

---

## What the App Handles For You (Do Nothing)

| Thing | How |
| ----------------------- | ---------------------------------------------------------------------------- |
| Agent wallet address | Generated by `python main.py init` or the "Generate Wallet" button in the UI |
| Agent private key | Same — written to your `.env` automatically |
| SQLite database | Created automatically on first run |
| LangGraph checkpointing | Automatic, uses the same `agora.db` |
| LLM provider detection | Auto-detected from which API key is present in `.env` |

---

## Required — App Will Not Work Without These

### 1. LLM API Key

The agent needs an LLM to reason about markets. Choose one of the three options below — only one block needs to be set in `.env`.

**Option A — Groq (recommended: free, fastest, best tool support):**

1. Go to [console.groq.com](https://console.groq.com)
2. Sign in → API Keys → Create Key
3. Copy the key (`gsk_...`)

**.env values to set:**
```
LLM_PROVIDER=groq
GROQ_API_KEY=gsk_...
LLM_MODEL=llama-3.3-70b-versatile
```

---

**Option B — NVIDIA NIM (free, access to large premium models):**

1. Go to [integrate.api.nvidia.com](https://integrate.api.nvidia.com)
2. Sign in → API Keys → Generate Key
3. Copy the key (`nvapi-...`)

**.env values to set:**
```
LLM_PROVIDER=nvidia
LLM_API_KEY=nvapi-...
LLM_MODEL=meta/llama-3.1-70b-instruct
```

---

**Option C — OpenAI or any OpenAI-compatible API (paid):**

1. Go to [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. Create a new key

**.env values to set:**
```
LLM_PROVIDER=openai
LLM_API_KEY=sk-...
LLM_MODEL=gpt-4o-mini
LLM_BASE_URL=https://api.openai.com/v1
```

> If `LLM_PROVIDER` is not set, the app auto-detects: Groq key present → Groq; `nvapi-` prefix → NVIDIA; otherwise → OpenAI.

---

### 2. Tavily API Key

Used by the scanner and researcher for web search.

1. Go to [app.tavily.com](https://app.tavily.com)
2. Sign up (free) → Dashboard → copy your API key (`tvly-...`)
3. Free tier: **1,000 searches/month** — plenty for the hackathon

**.env value to set:**
```
TAVILY_API_KEY=tvly-...
```

---

### 3. Arc Testnet RPC URL

Needed for onchain proof transactions and x402 payment verification.

1. Join the **Canteen Discord**
2. Find the Arc testnet RPC section — they provide a personal RPC URL
3. It looks like: `https://rpc.testnet.arc-node.thecanteenapp.com/v1/YOUR_KEY`

**.env value to set:**
```
ARC_RPC=https://rpc.testnet.arc-node.thecanteenapp.com/v1/YOUR_KEY
```

---

### 4. Arc Testnet USDC (Faucet)

Needed to send proof transactions and for users to pay x402 unlocks.

**After generating your wallet (step below):**

1. Copy your `AGENT_ADDRESS` from `.env`
2. Go to the Canteen Discord → faucet channel
3. Request testnet USDC to your address
4. A small amount (0.10 USDC) is enough for many proof txs + x402 unlocks

**How to generate the wallet first:**

```bash
# Option 1 — CLI
python main.py init

# Option 2 — Streamlit UI
streamlit run app.py
# Click "Generate Wallet" in the sidebar
```

## Optional — Circle Programmable Wallets

Circle is **not required** to run the app. It is used for wallet creation and balance reads. On-chain proof transactions always use web3.py directly because Arc testnet is not a Circle-supported chain.

If you want to set up Circle for future use or for judging purposes, a single command handles the full two-step setup:

```bash
python main.py circle-init
```

This registers your entity secret with Circle and creates a wallet set + wallet on Arc testnet, then prints the three values to add to `.env`. Requires `CIRCLE_API_KEY` and `CIRCLE_ENTITY_SECRET` to already be in `.env` — see `onchain/circle_setup/README.md` for instructions on where to get them.

## Full `.env` File — Filled In

```bash
# Copy .env.example to .env, then fill in:

# ── LLM — pick ONE option ─────────────────────────────────
# Option A (Groq — recommended)
LLM_PROVIDER=groq
GROQ_API_KEY=gsk_...
LLM_MODEL=llama-3.3-70b-versatile

# Option B (NVIDIA NIM)
# LLM_PROVIDER=nvidia
# LLM_API_KEY=nvapi-...
# LLM_MODEL=meta/llama-3.1-70b-instruct

# Option C (OpenAI)
# LLM_PROVIDER=openai
# LLM_API_KEY=sk-...
# LLM_MODEL=gpt-4o-mini
# LLM_BASE_URL=https://api.openai.com/v1

# ── Search ─────────────────────────────────────────────────
TAVILY_API_KEY=tvly-...           # from app.tavily.com

# ── Arc testnet ────────────────────────────────────────────
ARC_RPC=https://rpc.testnet.arc-node.thecanteenapp.com/v1/YOUR_KEY

# ── Auto-generated by `python main.py init` ───────────────
AGENT_ADDRESS=0x...
AGENT_PRIVATE_KEY=0x...

# ── Tier 2 intelligence (defaults — usually no change) ───
CLOB_ENABLED=true
DOMAIN_ROUTING_ENABLED=true
ENSEMBLE_ENABLED=false
RECENCY_DECAY_DAYS=30

# ── Tier 3 execution dry-run (defaults — no real CLOB orders) ─
EXECUTION_DRY_RUN=true
FALLBACK_BANKROLL=100
MAX_POSITION_FRACTION=0.10
MAX_TOTAL_EXPOSURE_FRACTION=0.35
MAX_THEME_EXPOSURE_FRACTION=0.20

# ── Defaults (no change needed) ───────────────────────────
MIN_EV_THRESHOLD=0.05
MIN_VOLUME=10000
TOP_N_PICKS=10
ARC_ENABLED=true
X402_PRICE_USDC=0.01
X402_ENABLED=true
```

## Launch Order

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Copy and fill in .env
cp .env.example .env
# edit .env with the keys above

# 3. Generate wallet (writes AGENT_ADDRESS + AGENT_PRIVATE_KEY to .env)
python main.py init

# 4. Fund wallet from Circle faucet (paste your AGENT_ADDRESS)

# 5. Run the app
streamlit run app.py

# OR use the CLI
python main.py scan
python main.py resolve    # update resolved outcomes in SQLite
python main.py metrics    # P&L, Brier, calibration summary
python main.py portfolio  # latest sizing + risk summary (after a scan)
```

Copy the full variable list from [.env.example](.env.example) — it includes Tier 2 and Tier 3 flags with comments.

## Checklist

- `GROQ_API_KEY` (or `LLM_API_KEY` for NVIDIA/OpenAI) — from your chosen LLM provider
- `TAVILY_API_KEY` — from tavily.com (free)
- `ARC_RPC` — from Canteen Discord
- Run `python main.py init` — generates wallet, writes to `.env`
- Fund wallet from Circle faucet (testnet USDC)
- *(Optional)* Circle setup — run `python main.py circle-init` (see `onchain/circle_setup/README.md`)
- *(Optional)* Tier 2 — set `ENSEMBLE_ENABLED=true` only if you have multiple LLM API keys (slower scans)
- Tier 3 runs in dry-run by default (`EXECUTION_DRY_RUN=true`) — no Polymarket trading keys required
